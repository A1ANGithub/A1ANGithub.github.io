水壶问题

预备知识

贝祖定理

问题

有两个容量分别为 x升 和 y升 的水壶以及无限多的水。请判断能否通过使用这两个水壶，从而可以得到恰好z升的水？
如果可以，最后请用以上水壶中的一或两个来盛放取得的z升水。
你允许：
装满任意一个水壶
清空任意一个水壶
从一个水壶向另外一个水壶倒水，直到装满或者倒空

示例

Solution Code

解析

每回水壶内水的变动为x或者y（x-y或者y-x等操作可以通过调整x和y的系数得到）。即证明是否存在整数a和b，使ax+by=z成立（题中要求x+y>=z）。根据贝祖定理，即证明z%gcd(x,y)==0是否成立（gcd求两个数的最大公约数）。

贝祖定理的个人证明

  • 证明充分性
    设gcd(x,y)=m,则x=am,y=bm,则存在系数c,d使cam+dbm=m，即cx+dy=m;若z%m==0,则z%(cx+dy)==0成立。

  • 证明必要性
    若z%(cx+dy)==0，则z%(cam+dbm)==0，则z%m==0

最长上升子序列

预备知识

  • 动态规划
  • 贪心算法+二分查找

    问题

    给定一个无序的整数数组，找到其中最长上升子序列的长度。

    示例

    Solution Code

    方法一

    解析

    时间复杂度为O(n^2)
    使用动态规划；定义dp[]数组，dp[i]为第i个数字结尾的最长上升子序列的长度；遍历到nums数组第i个数，在用j遍历0–i，若nums[i]>num[j]，把dp[j]+1赋值给dp[i]，直至遍历结束（其中dp[i]取最大值）。最后遍历dp[i]数组，最大值即为最长上升子序列。

方法二

解析

时间复杂度为O(nlogn)
使用贪心算法+二分查找；维护一个数组dp[]，首先讲nums[0]放入，初始长度为1；遍历nums，依次对nums[i]进行处理；若nums[i]大于dp末尾值，讲nums[i] append入dp；若nums[i]小于dp末尾值，则采用二分查找找到第一个比它小的值，并将nums[i]替代该值的下一位置的值（注意若都比nums[i]大，则放在首位）。

总结

发现第二种方法虽然时间复杂度低，但是略微费空间，以下原文链接有一位力扣成员使用C语言编写，空间时间均接近双百
